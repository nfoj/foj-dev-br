AWS - Cloud

Introdução

O que é AWS Cloud?

A AWS Cloud é uma plataforma de computação em nuvem fornecida pela Amazon Web Services (AWS).

Modelos:

Tradicional IT

Seu gerenciamento:

  • Aplicações;
  • Dados;
  • Tempo de execução;
  • Middleware;
  • Sistema operacional;
  • Virtualização;
  • Servidores;
  • Armazenamento;
  • Rede.

IaaS (Infraestrutura como Serviço)

Seu gerenciamento:

  • Aplicações;
  • Dados;
  • Tempo de execução;
  • Middleware;

Fornecido como serviço:

  • Sistema operacional;
  • Virtualização;
  • Servidores;
  • Armazenamento;
  • Rede.

Mais flexibilidade quanto à configuração de rede e armazenamento:

  • Gerencia aspectos de segurança;
  • Configura controles de acesso.

Exemplos:

  • AWS EC2;
  • AWS Elastic Block Storage (EBS);
  • AWS Virtual Private Cloud (VPC).

PaaS (Plataforma como Serviço)

Seu gerenciamento:

  • Aplicações;
  • Dados;

Fornecido como serviço:

  • Tempo de execução;
  • Middleware;
  • Sistema operacional;
  • Virtualização;
  • Servidores;
  • Armazenamento;
  • Rede.

Observações:

  • Não gerencia a infraestrutura subjacente;
  • A AWS gerencia o sistema operacional: aplicação de patches de banco de dados, configuração de firewall e recuperação de desastres;
  • O cliente gerencia o código e os dados.

Exemplos:

  • AWS Lambda;
  • AWS Relational Database Service (RDS);
  • AWS Elastic Beanstalk.

SaaS (Software como Serviço)

Fornecido como serviço:

  • Aplicações;
  • Dados;
  • Tempo de execução;
  • Middleware;
  • Sistema operacional;
  • Virtualização;
  • Servidores;
  • Armazenamento;
  • Rede.

Observações:

  • Hospedagem centralizada;
  • Modelo de assinatura ou pagamento por uso;
  • Acessado através de um navegador, aplicativo móvel ou API (Interface de Programação de Aplicativos);
  • Não há necessidade de gerenciar a infraestrutura subjacente que suporta o serviço.

Exemplos:

  • AWS Trusted Advisor;
  • AWS Shield;
  • AWS Chime.

Infraestrutura

Região (Zonas de Disponibilidade)

Área geográfica

  • A replicação de dados entre regiões é controlada pelo consumidor;
  • A comunicação entre regiões utiliza a infraestrutura de rede de backbone da AWS;

Observação:

  • A região fornece redundância completa e conectividade à rede;
  • A região normalmente consiste de duas ou mais Zonas de Disponibilidade.

Resumo:

  • Região: São Paulo, Virgínia, ...
  • Existe comunicação, mas você a configura;
  • Cada região é interdependente, funcionam de forma diferente;
  • Cada região possui um conjunto de data centers.

Como escolher uma região

  • Governança de dados: Os dados podem sair do país?
  • Proximidade dos clientes (latência);
  • Serviços disponíveis na região;
  • Custos (variam por região).

Uma Zona de Disponibilidade é um local discreto dentro de uma região que compreende um ou mais data centers, projetados para fornecer tolerância a falhas e redundância.

  • Elasticidade e escalabilidade;
  • Tolerância a falhas;
  • Alta disponibilidade.

Uma única instância de produto pode ser replicada em várias Zonas de Disponibilidade.
