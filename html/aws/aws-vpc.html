AWS VPC (Virtual Private Cloud)

Introdução

O que é?

Serviço de rede privada virtual que permite que você lance recursos da AWS em uma rede virtual isolada definida por você.

Seção Isolada Logicamente pode executar recursos da AWS em uma rede virtual que você mesmo define.

Controle sobre seus recursos de rede virtual:

  • Seleção do intervalo de endereços IP;
  • Criação de sub-redes;
  • Configuração de tabelas de rotas de gateway de rede.

Permite que você personalize a configuração de sua rede VPC.

Múltiplas camadas de segurança.

Resumo: uma rede principal e sub-redes dentro dela, que podem ser configuradas.

Tabela de Rotas

Um conjunto de regras ou rotas que direcionam o tráfego de rede do seu subnet.

  • Cada rota tem uma origem e um destino;
  • Toda tabela tem uma rota local para comunicação dentro do VPC;
  • Cada subnet deve estar associada a uma tabela de roteamento (máximo).
Destino Destino
10.0.0.0/16 Local

Resumo: uma tabela de roteamento é como um Waze para a internet.

Subnets

Subnets Públicos: Precisa ser publicamente acessível. Exemplo: o site de uma loja;

Subnets Privados: Deve ser acessível apenas da rede privada, como um banco de dados.

Internet Gateway (IGW)

Permite tráfego público da internet para o seu VPC.

Resumo: Atua como um gateway, permitindo o tráfego da internet pública para o seu VPC.

NAT

Diferente de um gateway, o NAT recebe dados de entrada da internet, mas não envia ativamente dados de saída para a internet, a menos que seja em resposta a uma solicitação de entrada.
